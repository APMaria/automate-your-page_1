Key Concepts

Unit 1 Basics of the Web

The web is made up of HTML (Hypertest Markup Language) and hyperlinks and was invented in 1990s with over 30 billion pages. HTML is made up of:

  1. Text (What you can see)
  2. Markup (What it looks like)
  3. References to other documents (e.g. images & video)
  4. Links to other pages

*Note - Plain text is plain text in HTML

HTML

HTML markups are made up of tags. Some tags have opening and closing tags. Others (Void tags) do not have closing tags. In HTML all "whitespace" (e.g. multiple lines) displays as a single line.

Note: Computers are stupid and will only do exactly what you tell them to!

  1. A Tag is an HTML that is always contained within angled brackets unless it it a "Void" tag.
  2. An element refers to everything within a set of opening and closing tags.
  3. An attribute is the property of an HTML element (e.g. href).

Tags Covered

Inline Tags include:

  • B Tag (bold)
  • EM Tag (italic)
  • IMG Tag (image) use src (source)
  • SPAN Tag

Block Tags create an invisible box around text and include:

  • P Tag (paragraph)
  • DIV Tag

HTML Attributes

HTML Attributes can point to a link. Use an anchor (A Tag) and href. Example: a href(attribute name)="www.udacity.com"(value) contents closing tag.

Key Concepts 2

Webpages, Documents and Structure

HTML is the structure and CSS (Cascading Style Sheets) is the design. There are rules for how websites behave. There is a tree-like structure (DOM - Document Object Model). Use the sideways arrows to drill down

Boxes to HTML

Designs should be broken down into boxes (e.g. a title, images, text, captions, etc.) Each box is identified with div.


Google Chrome Developer Tools display the HTML structure of the document. The Style box in the developer tools dictates what you see (i.e. color, size, borders) and can be altered via the syntax and rules).

Scratchpad vs Codepen vs Sublime Text 2

  1. Scratchpad = Immediate Response and Ease of Use when working with HTML and/or CSS.
  2. Codepen = 2-3 second response and is slightly more difficult to use but offers ease of sharing and supports Java as well as HTML and CSS.
  3. Sublime Text 2 = A need to refresh to view changes and is difficult to use until you are familiar with it. Supports ALL languages with a few exceptions.

Key Concepts 3

Adding CSS Style to HTML

You can have several CSS sheets.

You can enter code comments that can be viewed in the code but will not show up on the webpage. The format is different depending on if you are writing HTML or CSS.

Benefits of Avoiding Repetition

The primary benefit of adding CSS style to your HTML is to avoid repetition. There are several benefits to avoiding repetition:

Minimizes errors - type the rule once - one and done!
Reduces the amount of code you have to write - again one and done!
Promotes consistency - everything looks uniform

The Box Model

Four (4) areas: Margin, Border, Padding and Content. To calculate the total size, add the following in "Computed" view:

  • Left border
  • Left Padding
  • Content
  • Right Padding
  • Right Border

If you set the parameters by percentage, the contenet will display relative to that setting when the window is resized

To add an image, create a new folder and name it "Images" within your root folder.

Code, Test and Refine

Ask yourself:

  1. Have I defined the the natural boxes?
  2. Do I need a heading?
  3. Is the sizing of the boxes good?
  4. Is the positioning of the boxes good?
  5. Any other details?

Five Steps:

  1. Visually Boxify
  2. Look For Repeated Styles & Semantic Elements
  3. Write HTML
  4. Apply Styles (from biggest to smallest)
  5. Fix Things (check other browsers, resize windows, check until you are happy)

cs101: Building a Search Engine

What We Will Do

The programs we write will be Python code which we will input into another program which is a Python interpreter that follows the instructions in our code (procedural thinking) and does that by following instructions in its code. We will be able to run all using our web browser.

Note the demonstration of technological thinking (the entire previous paragrah) as well as abstract thinking (considering a Python program, a Python interpreter and a web browser as different versions of the same thing).

Terms to Know

  1. Web Crawler collects data from the web.
  2. Computer is a machine that can do everything; however, it can't do anything without a program.
  3. A program provides very precise instructions that tell a computer what to do.
  4. Programming Language (Python) enables the ability to write a program that a computer can understand without ambiguity.
  5. Recursive Definitions allow us to define infinitely many things using a few simple rules (expressions, operators).

Python Expressions

An expression is something that has a value. The expression non-terminal is on the left and can be replaced by an expression, followed by an operator, followed by another expression (e.g. 1 + 1).

Note: Python views a difference between "1" and "1.0". When you divide a number without a decimal by another number without a decimal, Python ignores the decimal part of the answer. In order to force Python to give you an answer including a decimal, you will need to make one of the numbers into a decimal by putting a decimal point after it (e.g. 5/2.0 = 2.5 and so does 5.0/2).

Variables and Strings

Python requires code be written with a very specific syntax. Small typographical errors can lead to BIG problems (e.g. 2 + 2 will output 4 and "2" + "2" will output 22).

One of the most important (and most confusing!) concepts in programming is the variable. In Python a variable is a name that refers to a value. That value can be be any sequence of letters, numbers and/or underscores; but, it cannot start with a number. Variables are useful in programming as they:

  • Afford code readability by using names that make sense to humans
  • Provide a way to store the values of important data
  • Allow flexibility by enabling the modification of values
In order to introduce a variable we have to give it an assignment. That "assignment" is known as an assignment statement.Name = Expression is an assignment statement. For example if I make the assignment that cycle_time = 40 cDays, I can then use "cycle-time" and it will always mean 40 cDays unless I change it. Note: In programming the equals sign means assignment (think of it as an arrow) and not equal to. It means that the value on right side evaluates to/is assigned to the variable of the left.

A string is a sequence of characters (can be a single word, letter or number) surrounded by quotes (either single or double as long as the same type is used on either end of the sequence). Strings can be concatenated, multiplied, indexed and extracted.

  • String Concatenation - string A + string B outputs the two strings together with no spaces
  • String Multiplication - string A * number N outputs that number of copies of string A pasted together with no spaces.
  • String Indexing - string A [Number N] outputs the position of the character/character string in string A. Positions are counted from start (left) at 0 or from stop (right - negative number).
  • String Extraction - string A [start number:stop number] outputs a subsequence of string A from start position to just before stop position. If start number is missing, start from beginning of string A. If stop number is missing, go to end of string A.
  • String Extraction (Find) - search string A.find (target string) outputs the position number where the target string first appears. For subsequent occurrences use: search string A.find(target string, start number) where the start number is the position of the first appearance.

Inputs and Functions

A function takes input (parameters) and processes that input (also known as "operand" also know as "argument") to produce an output (e.g. when x is true; do what I tell you to do with x and produce y).

A function can have any number of inputs; however, the number of inputs must match what the function expects.

A function once created can be reused forever; thus, avoiding the need for repetition!

The format for making a function starts with a line of code beginning with "def" followed by the parameters in parentheses.

The format for using a function is a line of code beginning with an action (e.g. "print") followed by the name of the function and a value(s) in parentheses. A function also requires a return statement that tells the function what the output should be. If there is no return statement, the output will be "None".

Control Flow and Loops: If and While

While Loops let you write code that does the same thing over and over - any number of times - as long as the test expression is true.

A "break" gives us a way to stop a loop even while the test expression is true.

If and Else functions check to see if the test expression is true and will evaluate in accordance with the code.

Making decisions typically includes comparisons. In Python we use several number operators. The format: number, comparison operator, number will yield a boolean value (True or False) and NOT a number.

To use an "If" function, we type "if" followed by a test expression followed by a colon (don't forget the colon!). The "block" will evaluate if the instance is true and will not NOT evaluate if the instance is false. If the block is not indented, it will execute whether or not the test expression is true.

When using the "Or" operand, the following rules apply:

  1. If the first expression evaluates to "true", the value is true and the second expression is not evaluated.
  2. If the first expression evaluates to "false", the value is the value of the second expression.

Debugging Strategies

  1. Examine error messages when programs crash.
    • The last line of the Python traceback will tell you what went wrong. Reading backwards from there will tell you more about where the problem occurred.
  2. Work from example code.
    • If your modified code doesn't work, comment it out and do step-by-step modifications to the example code until it does what you want.
  3. Make sure examples work.
    • Just because you find example code doesn't mean it will work in your system. Check the example code you're using to make sure it behaves the way you expect.
  4. Check (print) intermediate results when you're code doesn't crash, but doesn't behave as expected.
    • Add print statements to your program to see where in the code things stop behaving correctly.
  5. Keep and compare old versions.
    • When you have a working version of your code, save it before you add to the code. This will give you something to go back to if you introduce too many new bugs.

Structured Data: Lists and For Loops

In Modular code the "data" is separated from its visual presentation so you can change one without worrying about the other. To make this possible keep the raw text of notes (data) in one place, the structure (HTML) and the visual (CSS) somewhere else.

Systems Thinking requires the utilization of five key steps:

  1. Understand the requirements of the problem. Inputs? Outputs?
  2. Plan an approach to solving the problem.
  3. Re-familiarize yourself with code you've already written/learned that may help.
  4. Write new code.
  5. Put all the pieces together.

Structured data can be a string (sequence of characters) and/or a list (sequence of anything).

A nested list can be a list within a list and can include both strings and integers. When using a long list in python interpreter, make separations with commas.

Lists support mutation which enables us to change the value of the list without creating a new list object. Strings do NOT support mutation.

Aliasing is two ways to refer to the same object (i.e. a person's government name and their nickname refer to the same person).

List Operations include:

  • Append (works similar to "find") and adds an element to an existing list.
  • Plus concatenates and does NOT add.
  • len is short for length and counts the elements in a list or string.

For Loops take the format: for a variable we introduce in a specified list do what we instruct.

The Index Method takes a value in a list and returns the first position where that value is found in the list and otherwise returns an error.

How to Solve Problems

Five Steps:

  1. What are the inputs?
  2. What are the outputs?
  3. Work through some examples by hand. You should know what your expected output should be and what it should look like.
  4. Develop a basic mechanical solution to use as a starting point. This solution may include pseudo code and may NOT be 100% correct and that's okay. Use it to get you part of the way to your ultimate solution. Use stub procedures and helper functions as applicable. Use comments to note any inaccuracies and don't forget to test!
  5. Develop incrementally - in small steps and test as you go. This will enable you to identify and correct errors in your code easier and more methodically.
